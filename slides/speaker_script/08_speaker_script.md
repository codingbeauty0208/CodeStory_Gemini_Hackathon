### Slide 8
The API layer defines how external systems and users interact with our "Traveller" application. All the travel-related endpoints are organized under the `/api/travels` base path, providing a clear and consistent URI structure. We implement a full suite of RESTful endpoints: `GET` requests to retrieve all travel records or a specific one by its ID, `POST` to create new travel entries, `PUT` to update existing records, and `DELETE` to remove them. This adherence to REST principles makes the API intuitive and easy to consume. For both requests and responses, the application is designed to communicate using JSON, which is the industry standard for web service data exchange due to its lightweight and human-readable nature.
