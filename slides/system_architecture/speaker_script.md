## System Architecture

Slide 1:
"The Traveller application is built upon the well-established Model-View-Controller, or MVC, architectural pattern, which is standard in Ruby on Rails development. This structure, evident in our `app/` directory, cleanly separates concerns, making the codebase more maintainable and scalable. Our **Models**, residing in `app/models`, are responsible for encapsulating the application's business logic and managing interactions with the database. These include `User`, `Trippackage`, `Company`, `Slot`, and `Feedback` models, which define data structures and their relationships. The **Views**, located in `app/views`, handle the presentation layer. They are primarily composed of ERB templates that generate dynamic HTML, with `Jbuilder` templates used for rendering JSON responses. Lastly, the **Controllers**, found in `app/controllers`, act as intermediaries. They receive and process user input, interact with the appropriate models to retrieve or manipulate data, and then select the correct view to render the final response back to the user. This clear separation ensures a modular and organized application structure."

Slide 2:
"For its user-facing components, Traveller employs a modern frontend asset management architecture, which is critical for delivering a responsive and engaging user experience. The application leverages Rails' robust asset pipeline, managed by `sprockets-rails`, to efficiently handle static assets such such as images, stylesheets, and JavaScript files. For JavaScript, we’ve integrated modern bundling capabilities through `jsbundling-rails`, which utilizes `esbuild`. This is evident from our `package.json` scripts, configured to compile JavaScript from `app/javascript` into `app/assets/builds`. Similarly, for styling, `cssbundling-rails` is integrated with `sass`. Our `package.json` contains a `build:css` script that compiles SCSS stylesheets, specifically `application.bootstrap.scss`, into a consolidated `application.css` file. These tools allow us to write maintainable, modular frontend code that gets optimized for production. Furthermore, the application extensively uses `@hotwired/turbo-rails` and `@hotwired/stimulus` to provide a dynamic, single-page-application-like feel with minimal JavaScript, ensuring fast page loads and smooth transitions, complemented by the Bootstrap 5.2.3 framework and Bootstrap Icons for a consistent and modern user interface."
