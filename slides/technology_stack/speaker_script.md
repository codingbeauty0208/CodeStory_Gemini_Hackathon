## Technology Stack

Slide 1:
"The foundation of Traveller's backend is built upon a robust and widely-adopted technology stack, starting with the **Ruby programming language**. Specifically, the project utilizes Ruby version `3.1.3`, as explicitly declared in the `.ruby-version` file and reaffirmed in the `Gemfile`. This ensures a consistent and stable environment for development and deployment. At its core, the application leverages the **Ruby on Rails web framework**, operating on version `~> 7.0.4, >= 7.0.4.3`, as detailed in the `Gemfile`. Rails provides the comprehensive MVC structure and conventions that underpin the entire application. For serving web requests, **Puma** is configured as the default web server, specified as `gem "puma", "~> 5.0"` in the `Gemfile` and further configured in `config/puma.rb`. This choice offers performance and concurrency suitable for modern web applications. Finally, for persistent data storage, **MySQL** has been selected as the relational database. This is indicated by the presence of the `mysql2` gem, version `~> 0.5`, in the `Gemfile`, and its connection parameters, including database names and user credentials, are meticulously defined within the `config/database.yml` file for different environments."

Slide 2:
"Beyond its core languages and frameworks, Traveller benefits from a suite of carefully selected libraries and gems that extend its functionality and enhance the user experience. For robust **user authentication and authorization**, the `Devise` gem, version `4.9.2`, is integrated. This powerful solution handles user registration, login, password recovery, and session management, providing a secure and configurable access control layer. To facilitate efficient **administrative tasks**, `rails_admin`, version `3.1.2`, is implemented. This gem provides a comprehensive and customizable administrative interface, accessible via the `/admin` route, allowing for seamless management of all database resources without custom development. On the frontend, `turbo-rails` and `stimulus-rails` are key components of the **Hotwire** suite. They deliver a dynamic, highly interactive user experience by minimizing full page reloads and enhancing client-side interactivity, creating an almost SPA-like feel. For **asset compilation and management**, `jsbundling-rails` (leveraging `esbuild`) and `cssbundling-rails` (with `sassc-rails`) are used to efficiently process JavaScript and CSS assets. Lastly, `jbuilder` is included to simplify the **construction of JSON API responses**, ensuring that data can be cleanly exposed to other services or frontend components if needed. For **testing**, `Capybara`, `selenium-webdriver`, and `webdrivers` are used for comprehensive system testing, enabling automated browser interactions to validate end-to-end functionality."
